# Ollama CLI

## 基本情報

| 項目 | 内容 |
|------|------|
| 種別 | CLI（ローカル LLM ランナー） |
| ログ形式 | プレーンテキスト（コマンド履歴のみ） |
| ローカルログ | 限定的 |
| リポジトリ | [ollama/ollama](https://github.com/ollama/ollama) |

## 概要

Ollama はローカルで LLM を実行するためのオープンソースツール。CLI にはインタラクティブなチャットモードがあるが、**会話履歴の自動的なディスク保存は行わない**。保存されるのはコマンド入力履歴（矢印キーでの呼び出し用）のみ。完全な会話ログにはサードパーティクライアントやシェルリダイレクションが必要。

## 保存場所

| OS | パス |
|----|------|
| macOS | `~/.ollama/` |
| Linux | `~/.ollama/`（ユーザー）または `/usr/share/ollama/.ollama/`（システムサービス） |
| Windows | `C:\Users\<username>\.ollama\` |

## ファイル構造

### クライアント側（実行ユーザー）

```
~/.ollama/
├── history                    # コマンド入力履歴（readline、会話ログではない）
```

### サーバー側（ollama ユーザーまたは実行ユーザー）

```
~/.ollama/                     # または /usr/share/ollama/.ollama/（Linux systemd サービス時）
├── id_ed25519                 # 秘密鍵
├── id_ed25519.pub             # 公開鍵
├── server.json                # サーバー設定
└── models/
    ├── manifests/             # モデルマニフェスト
    └── blobs/                 # モデルバイナリデータ (sha256-*)
```

Linux で systemd を使用する場合、サーバーは `ollama` ユーザー（ホーム: `/usr/share/ollama/`）で実行される。クライアント側の `~/.ollama/history` は呼び出しユーザーのホームディレクトリに保存される。

## 保存されるもの・されないもの

### 保存される: コマンド履歴 (`~/.ollama/history`)

CLI 入力の readline 履歴ファイル（上下矢印キーでの入力呼び出し用）。保存されるのは**ユーザーの入力テキストのみ**で、モデルの応答は含まれない。

- デフォルト上限: 100 エントリ（設定可能）
- 自動保存はデフォルトで有効
- `/set nohistory` で無効化可能

### 保存されない: 会話履歴

完全な会話内容（ユーザープロンプト＋モデル応答）は**ディスクに自動保存されない**。会話コンテキストはアクティブなセッション中のメモリ内にのみ存在する。

## セッション管理コマンド

| コマンド | 説明 |
|---------|------|
| `/save <name>` | 現在のセッションを新しいモデルとして保存（会話＋システムメッセージ） |
| `/load <model>` | 別のモデルを読み込み |
| `/clear` | 会話コンテキストをクリア |
| `/set history` | コマンド履歴を有効化 |
| `/set nohistory` | コマンド履歴を無効化 |
| `/show info` | モデル詳細を表示 |
| `/show system` | システムメッセージを表示 |

### /save コマンドの動作

`/save <name>` は現在のモデルを継承し、会話履歴とシステムメッセージをセッションスナップショットとして埋め込んだ**新しいモデル**を作成する。ログファイルではなく、Ollama のモデルストレージに保存されるモデル成果物。

## 環境変数

| 変数 | 説明 |
|------|------|
| `OLLAMA_MODELS` | カスタムモデル保存ディレクトリ |
| `OLLAMA_HOST` | サーバーバインドアドレス（デフォルト: `127.0.0.1:11434`） |
| `OLLAMA_DEBUG` | デバッグログの有効化（プロンプトがサーバーログに含まれる場合がある） |

## 会話ログの代替手段

Ollama はネイティブに会話を保存しないため、以下の方法が一般的:

1. **シェルリダイレクション**: `ollama run model 2>&1 | tee chat.log`
2. **サードパーティクライアント**: Open WebUI、Ollama GUI 等（各々が独自のストレージを実装）
3. **API ベースのログ**: REST API (`/api/chat`) 経由でリクエスト/レスポンスをキャプチャ
4. **デバッグモード**: `OLLAMA_DEBUG=1` でプロンプトをサーバーログに含める

## 注意事項

- Ollama は主にローカル LLM ランナーであり、AI コーディングエージェントではない
- `/save` コマンドは会話状態をモデルとして保存し、ログファイルとしてではない
- デバッグモード有効時のサーバーログにはプロンプトが含まれうるが、構造化された取得には不向き
- Ollama 上に構築されたサードパーティ UI は独自の会話永続化を実装している
